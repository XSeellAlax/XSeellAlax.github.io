外观模式

外观模式是一种使用频率高的结构型设计模式，通过引入一个外观角色来简化客户端与子系统之间的交互，即提供一个统一访问子系统的接口，客户端通过该接口来访问其他子系统，从而避免客户端直接与子系统的交互，减少双方之间的耦合度，客户端使用起来更方便。

外观模式的定义

外观模式：为子系统中的一组接口提供一个统一的入口。外观模式定义了一个高层接口，这个接口使得这一子系统更加容易使用。

简单工厂模式

为什么要学设计模式

C++ 中的设计模式可帮助开发人员创建可维护、灵活且易于理解的代码。它们浓缩了经验丰富的软件架构师和开发人员的专业知识和经验，使新程序员更容易遵循既定的最佳实践。

简单工厂模式实例

在本教程中，您将学习如何使用 C++ 实现简单工厂模式。简单工厂模式是一种创建对象的模式，它提供了一个静态工厂方法，该方法返回指定类的实例。
步骤：

  1. 创建一个接口。
  2. 创建一个类，该类实现接口。
  3. 创建一个类，该类包含一个静态工厂方法，该方法返回接口的实现类的实例。
  4. 创建一个类，该类包含一个构造函数，该构造函数初始化接口的实现类的实例。

设计模式及其C++实现

引言

设计模式是软件设计中常见问题的典型解决方法。每个模式就像一张蓝图，你可以通过对其进行定制来解决代码中的特定设计问题。

开始学习

Factory Method

工厂方法模式:使用工厂创建产品，而不需要知道产品创建的具体细节

在父类中提供一个创建对象的接口以允许子类决定实例化对象的类型。

Abstract Factory

抽象工厂模式: 提供一个接口，用于创建相关或依赖对象的家族，而不需要明确指定具体类。

Builder

建造者模式: 将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

Adapter

适配器模式: 将一个类的接口转换成客户期望的另一个接口。适配器让原本接口不兼容的类可以合作无间。

Singleton

单例模式: 保证类只有一个实例，并提供全局访问的节点

Prototype

原型模式: 允许对象复制自身，而不依赖自身代码
用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象

Bridge

桥接模式: 将抽象部分与它的实现部分分离，使它们都可以独立地变化

Fiter

过滤器模式: 允许开发者使用不同的标准来过滤对象

Composite

组合模式“: 将对象组合成树形结构以表示“部分-整体”的层次结构。COmposite使得用户对单个对象和组合对象地使用具有一致性。

Decorator

装饰器模式：动态地给一个对象添加一些额外的职责。

Facade

外观模式: 为子系统中的一组接口提供一个一致的界面，Facade模式定义了一个高级接口。，这个接口使得这一子系统更加容易使用。

Flyweight

享元模式: 运用共享技术有效地支持大量细粒度对象

Proxy

代理模式: 为其他对象提供一种代理来代替对该对象的访问

Chain of Responsibility

责任链模式: 将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。

Command

命令模式: 将一个请求封装成一个对象，从而可以使用不同的请求对客户端进行参数化；对请求派对或记录请求日志、以及支持可撤销的操作。

Interpreter

解释器模式: 给定一个语言，定义它的一种文法表示，并定义一个解释器，这个解释器使用该表示来解释语言中的句子。

Iterator

迭代器模式: 定义一种顺序访问聚合内每个元素的方法,而不暴露该对象的内部表示

Mediator

中介者模式: 定义一个对象来封装一系列的对象交互。使得这些对象交互时不必相互显式引用，从而使其耦合松散，而且可以独立地改变它们之间地交互。

Memento

备忘录模式: 在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态。

Observer

观察者模式: 定义对象间的一种一对多的依赖关系，当一个对象发生改变时，其他依赖于它的对象都得到通知并自动更新。

怎么做: 定义一个Array存储所有依赖的观察者

State

状态模式: 允许一个对象在其内部状态改变时改变

Strategy

策略模式: 定义一系列算法，将每一个算法封装起来，并让它们可以相互替换。策略模式让算法可以独立于使用它的客户端变化。

Hard Coding(硬编码)

Template

模板方法模式: 定义一个操作中算法的框架，而将一些步骤延迟到子类中。模板方法模式使得子类不改变一个算法的结构即可重定义该算法的某些特定步骤

Vistor

访问者模式: 在不修改已有代码的基础上，在已有类层次增加访问方式

MES: 制造执行系统

此时我们发现，图片被拉伸填充满整个画面，这使得原图变得失真，我们可以通过设置Image的fillModel来改变图片的填充显示格式，可取值如下:

Model 作用
Image.Stretch 图片拉伸自适应；（默认的）
Image.PreserveAspectFit 按比例缩放，不裁剪
Image.PreserveAspectCrop 均匀缩放，必要时裁剪
Image.Tile 像贴瓷砖一样
Image.TileVertically 水平拉伸，垂直平铺
Image.TileHorizontally 垂直拉伸，水平平铺
Image.Pad 原始图像不做处理

我们将图像的fillMode值设置为 Image.PressveAspectFit,这样图像就能按比例适应窗口了。
